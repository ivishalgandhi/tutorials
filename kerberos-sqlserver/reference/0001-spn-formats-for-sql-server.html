Remember

  • SPNs must be unique across the entire AD forest.
  • SPNs are tied to the account running the service, not necessarily the computer object.
  • Use FQDNs in SPNs and connection strings. IP addresses do not work for Kerberos.
  • The practical limit for SPNs on one AD object is about 1200 entries.
  • PostgreSQL uses a keytab file, not AD SPN registration, but the same FQDN-matching rule applies.
