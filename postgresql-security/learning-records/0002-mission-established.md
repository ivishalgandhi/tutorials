# Mission Established — End-to-End PostgreSQL Security Mastery

User manages a production PostgreSQL environment and wants to master its security model completely. The mission is pragmatic (not academic): harden a live production cluster, design role hierarchies, configure LDAP/AD, enable pgaudit for compliance, and be able to recite the 15 critical commands from memory.

**Implications:** All 11 lessons are in scope. Lessons should be ordered as specified (authentication → roles → RLS → schema → SSL → audit → checklist). Priority sequence mirrors how a DBA would actually harden a new cluster from scratch.
