Webhook Patterns

Production checklist for receiving webhooks in Go. Use this as a quick reference when building or reviewing a webhook receiver.

1. Verify the Signature First

Read the raw body, compute the HMAC, and compare with constant-time comparison. Never parse JSON before verifying. 

func verifySignature(body []byte, secret, sig string) bool {
    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write(body)
    expected := hex.EncodeToString(mac.Sum(nil))
    provided := strings.TrimPrefix(sig, "sha256=")
    return hmac.Equal([]byte(expected), []byte(provided))
}

2. Bound the Body

Always cap the body size to prevent memory exhaustion from a malicious sender. 

body, err := io.ReadAll(io.LimitReader(r.Body, 1<<20)) // 1 MB
if err != nil {
    http.Error(w, "bad request", http.StatusBadRequest)
    return
}

3. Acknowledge Asynchronously

Return HTTP 200 quickly and do the real work in a goroutine or queue. If the queue is full, return 503 so the provider retries. 

w.WriteHeader(http.StatusOK)

select {
case queue <- event:
default:
    log.Println("queue full; returning 503")
    http.Error(w, "overload", http.StatusServiceUnavailable)
}

4. Make It Idempotent

Store the event ID on the first delivery. Subsequent retries with the same ID are no-ops. 

_, err := db.ExecContext(ctx,
    `INSERT INTO events (id, payload) VALUES ($1, $2)
     ON CONFLICT (id) DO NOTHING`,
    eventID, payload)

5. Defend Against Replays

Reject payloads with a timestamp outside a short window (e.g., 5 minutes) unless you also store every event ID. 

age := time.Since(time.Unix(ts, 0))
if age < 0 || age > 5*time.Minute {
    http.Error(w, "stale webhook", http.StatusUnauthorized)
    return
}

Production Checklist

Concern Practice
Authentication HMAC-SHA256 signature with a shared secret; constant-time comparison.
Payload safety io.LimitReader to cap body size; read raw bytes before JSON parsing.
Reliability Return 200 quickly; process asynchronously; make handlers idempotent.
Retries Expect duplicate deliveries; use event ID or idempotency key to deduplicate.
Observability Log every receipt, verification failure, and processing error with structured fields.
Shutdown Drain the in-flight queue on SIGTERM; do not accept new HTTP requests during shutdown.
