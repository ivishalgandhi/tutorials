# Course Scope and Learning Profile

We started with the idea of extending an existing Go course, but the blog post topic — coding agents, concurrency, and webhooks — is distinct enough that a dedicated course keeps the mission tight. The learner is new to Go (some syntax, not idioms) and wants real-world examples to support interview conversations about the blog post.

Key implications for future sessions:
- Keep every lesson grounded in the blog post's agent → concurrency → webhooks arc.
- Explain every non-obvious Go construct; do not assume idiomatic fluency.
- Include a short exercise and cite primary sources / popular videos in every lesson.
- The course should remain standard-library-only so the learner can explain it without external dependencies.

## Mission shift

Original thought: extend an existing course.  
Resolved: create a new course, `go-agents-concurrency-webhooks`, because none of the existing courses match the blog post's focus.
