Claude Agent SDK: Subagents

The One Frame

One Frame
Prompts are probabilistic. Hooks are deterministic.
Subagents are isolated. The coordinator is responsible for decomposition, partitioning, and context passing.

Hub-and-Spoke Topology

  • Coordinator sits in the center; all subagents radiate out.
  • All inter-subagent communication routes through the coordinator — never agent-to-agent.
  • The coordinator decomposes, delegates, aggregates, validates, and handles errors.

Subagent Context Isolation

  • Subagents do not inherit the coordinator's conversation history.
  • Subagents do not inherit the coordinator's system prompt.
  • Subagents do not inherit parent tool results or skills (unless explicitly listed).
  • Every byte the subagent needs must be in its invocation prompt.
Memory aid
Subagent state: nothing carries over. The prompt is the universe.

The Task Tool

  • Coordinator spawns subagents via the Task tool.
  • Coordinator's allowedTools must include "Task" — otherwise it will never delegate.
  • Multiple Task calls in one coordinator response run concurrently.
  • Subagents cannot spawn their own subagents — keep orchestration one level deep.

Task Decomposition Patterns

PatternWhenExam signal
Fixed sequential pipelineStable structure; each step consumes the prior step's output"Every X follows the same flow"
Dynamic decompositionVarying requests; typed subagents (search / doc / synth)"Same roles, scope varies per query"
Orchestrator–workerUniform workers differentiated only by prompt-supplied scope"N workers, each a different sub-topic"
Evaluator–optimiserExplicit checkable quality criteria; first pass often inadequate"Iterate until coverage/format passes"
Evaluator–optimiser trap
The evaluator–optimiser pattern fails when criteria aren't objectively checkable. Subjective criteria → loop spins indefinitely.

"Won't delegate"Coordinator's allowedTools missing "Task"

Principle of Least Privilege

  • Give each subagent only the tools it needs for its scoped role.
  • Simple fact-checks can be delegated to a scoped verify_fact tool.
  • Complex multi-step research stays routed through the coordinator.

Multi-Agent Tradeoffs

BenefitCost / Risk
Parallel exploration; isolated context windows~15× token cost vs. single-agent chat
Specialized prompts and tool restrictions per subagentCoordination complexity; failure propagation must be designed
90% performance improvement on research-heavy tasksPoor fit for tasks with many interdependencies or few parallel subtasks

What NOT to do

  • Direct agent-to-agent communication.
  • Return empty results marked as success.
  • Forward the full conversation transcript to a human reviewer.
  • Retry every error type with the same delay.
  • Speculatively cache data "in case the synthesis agent needs it."
