# pgx v5 — The Scanner's Native Driver

Lesson 6 completed (final lesson). database/sql is wrong for a scanner: rows.Scan() requires pre-declared destination variables (breaks TOML-driven dynamic queries), DSN strings make GUC setting awkward, error types are opaque.

pgx advantages: rows.Values() returns []any — any query, any columns, no compile-time type knowledge. *pgconn.PgError exposes .Code (SQLSTATE), .Message, .Detail, .Hint. pgx.ParseConfig("") + struct field assignment replaces DSN hacks. RuntimeParams map sets GUCs cleanly.

Core scanner connect pattern: pgx.ParseConfig(""), set Host/Port/User/Password/Database, set ConnectTimeout, set RuntimeParams for statement_timeout/lock_timeout/application_name/idle_in_transaction_session_timeout, call pgx.ConnectConfig(ctx, config).

rows.Values() pattern: conn.Query() → rows.FieldDescriptions() for column names → rows.Values() for []any values → zip into map[string]any. This is what makes TOML-driven queries possible.

Migration from database/sql: go get pgx/v5, replace sql.Open with ConnectConfig, replace QueryContext with Query, replace rows.Scan(&a,&b) with rows.Values() + map. Four steps, no big-bang rewrite needed.
