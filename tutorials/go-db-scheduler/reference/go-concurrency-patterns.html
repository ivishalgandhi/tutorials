Quick reference for the concurrency primitives used in the scheduler. Source: Effective Go — Concurrency and Go Blog: Pipelines and Cancellation.

Buffered Channel as Semaphore

Limit how many goroutines can do something simultaneously.

// Create: capacity = max concurrent permits
sem := make(chan struct{}, maxConcurrent)

// Acquire: send blocks if channel is full
sem <- struct{}{}

// Release: receive frees one slot
defer func() { <-sem }()

// Per-server semaphores: one semaphore per server
type ServerSemaphores struct {
    mu   sync.Mutex
    sems map[string]chan struct{}  // key = serverID
}

func (ss *ServerSemaphores) acquire(serverID string, cap int) chan struct{} {
    ss.mu.Lock()
    defer ss.mu.Unlock()
    if _, ok := ss.sems[serverID]; !ok {
        ss.sems[serverID] = make(chan struct{}, cap)
    }
    return ss.sems[serverID]
}

Worker Pool

Fixed N goroutines drain a shared job channel. Prevents unbounded goroutine spawning.

func startWorkerPool(ctx context.Context, jobs chan *ScheduledJob, n int) {
    var wg sync.WaitGroup
    for i := 0; i < n; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {   // exits when jobs channel closes
                dispatch(ctx, job)
            }
        }()
    }
    wg.Wait()  // blocks until all N workers exit
}

Graceful Shutdown

On SIGTERM: stop new work, drain in-flight, then exit cleanly.

func main() {
    // 1. Tie a context to OS signals
    ctx, stop := signal.NotifyContext(
        context.Background(),
        syscall.SIGTERM, syscall.SIGINT,
    )
    defer stop()

    jobs := make(chan *ScheduledJob, 500)

    // 2. Start workers in background
    go startWorkerPool(ctx, jobs, 100)

    // 3. Block until signal
    <-ctx.Done()

    // 4. Stop new jobs: close channel so workers drain
    close(jobs)

    // 5. Drain with timeout (don't kill a running backup)
    drainCtx, cancel := context.WithTimeout(
        context.Background(), 5*time.Minute,
    )
    defer cancel()
    waitWithContext(drainCtx, &wg)
}
GOTCHA

Never pass the signal-cancelled context to the drain timeout — it is already cancelled. Create a fresh context.Background() for the drain phase, then wrap it in context.WithTimeout.

Context Propagation

A parent context cancels all children automatically.

parent, cancel := context.WithTimeout(ctx, job.Timeout)
defer cancel()  // always cancel to free resources

// Pass parent to the executor — it inherits the timeout
result, err := executor.Execute(parent, job)

// Check why it ended:
if errors.Is(ctx.Err(), context.DeadlineExceeded) {
    // job timed out — mark StatusTimeout
}
if errors.Is(ctx.Err(), context.Canceled) {
    // scheduler is shutting down — mark StatusCanceled
}

Select — Reacting to Multiple Channels

Wait for whichever channel fires first.

select {
case job := <-jobs:
    dispatch(job)          // new work arrived
case <-ctx.Done():
    return                 // shutdown signal
case <-ticker.C:
    generateAndEnqueue()   // periodic cron tick
}
